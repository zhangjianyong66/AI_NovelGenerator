# 项目功能地图与前端验收剧本

## Goal

为不熟悉原项目的维护者补齐一份“项目功能地图 + 前端验收剧本”，说明 AI Novel Generator 的核心能力、旧 Python GUI 与新 Tauri/Vue 前端的边界、真实生成主流程、文件产物、以及可重复执行的前端冒烟验收步骤。

用户价值：维护者不需要先读完整代码或理解所有历史功能，也能知道项目该怎么用、哪些功能当前是真实可用、哪些功能只是新前端壳层/待接入能力，并能按清单完成基础验收。

## Confirmed Facts

- 旧入口是 `python main.py`，启动 Python `customtkinter` 桌面 GUI。
- 新前端位于 `frontend/`，是 Tauri 2 + Vue 3 + TypeScript + Vite 工程，与旧 GUI 并行存在。
- 新前端默认通过 `frontend/src/services/serviceBridge.ts` 访问本地 FastAPI：`http://127.0.0.1:8000`。
- 本地 API 入口为 `app/api/server.py`，已覆盖项目配置、模型设置、核心项目文件、章节、生成任务、知识工具、角色库和 WebDAV 配置。
- `frontend/README.md` 明确：生成任务接口当前返回排队状态和日志，尚未接入真实 LLM 执行器。
- README 描述的真实小说生成主流程是：配置模型与小说参数 -> 生成设定 -> 生成目录 -> 生成章节草稿 -> 定稿当前章节 -> 可选一致性审校。
- 常见输出文件包括 `Novel_setting.txt`、`Novel_directory.txt`、`outline_X.txt`、`chapter_X.txt`、`global_summary.txt`、`character_state.txt`、`plot_arcs.txt` 和 `vectorstore/`。
- 新前端页面包括：项目、工作台、章节编辑、生成任务、知识库、设置。
- 当前仓库已有前端 UI 重构任务归档，UI 组件和页面重构基本完成；本任务不继续扩大 UI 重构范围。

## Requirements

- 产出一份面向维护者/新接手开发者的中文文档，优先帮助“不知道原项目功能的人”理解项目。
- 文档定位为开发者/维护者验收材料，但使用新手能看懂的语言；重点不是完整终端用户手册，而是帮助维护者判断功能边界、使用路径和验收方法。
- 文档必须区分“旧 GUI 真实完整能力”和“新前端当前已接入/未接入能力”，避免误导验收。
- 文档必须把前端验收动作按风险分级，至少区分：
  - 只读检查：打开页面、读取配置、读取项目文件、读取章节、查看任务日志。
  - 可逆写入：保存临时输出目录、保存测试文本、导入测试知识文件、导入/保存测试角色。
  - 谨慎/破坏性操作：清空 `vectorstore/`、WebDAV 恢复配置、覆盖已有章节或项目文件。
- 文档必须要求冒烟验收使用临时输出目录和测试文件，避免污染真实小说工程。
- 文档必须给出项目核心功能地图，至少覆盖：
  - 模型与 Embedding 配置
  - 小说参数与输出目录
  - 小说设定生成
  - 章节目录生成
  - 章节草稿生成
  - 章节定稿与状态更新
  - 一致性审校
  - 核心文件编辑
  - 知识库与向量库
  - 角色库
  - WebDAV 备份/恢复
- 文档必须说明主要文件产物的用途和生成/编辑来源。
- 文档必须提供不依赖真实 API Key 的前端冒烟验收剧本，重点验证配置、文件读写、页面状态和知识/角色文件链路。
- 无 API Key 冒烟验收不得要求真实 LLM、真实 Embedding、真实 WebDAV 服务或真实远程账号。
- 无 API Key 冒烟验收必须说明生成任务页的预期结果：任务只进入 `queued` 状态，日志包含“等待执行器接入”或等价信息，这不是生成成功。
- 文档必须提供依赖真实 LLM/Embedding 的完整功能验收剧本，明确这是旧 GUI 或未来真实生成接入后的验收范围。
- 文档必须说明本地启动方式：
  - 旧 GUI：`python main.py`
  - 新前端联调：`./scripts/dev.sh`
  - 或后端 `uvicorn app.api.server:app --reload --host 127.0.0.1 --port 8000` + 前端 `cd frontend && npm run dev`
- 文档必须给出“验收前准备数据”建议，例如临时输出目录、示例主题、示例章节数、示例角色文件/知识文件。
- 文档必须包含一张“功能状态矩阵”，按功能列出：
  - 用户目标
  - 旧 GUI 是否完整支持
  - 新前端当前是否真实支持
  - 本地 API 边界
  - 冒烟验收方法
- 文档必须包含一张“输出文件速查表”，列出文件/目录、用途、由哪个流程生成、能否在新前端编辑、验收注意事项。
- 文档必须包含“安全验收约定”，明确不得提交或泄露 `config.json`、API Key、真实账号、私有 Base URL、真实生成正文和向量库数据。
- 文档应放在 `docs/` 下，文件名表达清楚用途，并在必要时从 `frontend/README.md` 或根 README 引导到该文档。
- 如实施中发现新的项目运行约定，按项目规则更新根目录 `AGENTS.md`。

## Out of Scope

- 不在本任务中接入真实 LLM 生成执行器。
- 不重构前端 UI、不调整页面布局、不新增业务功能。
- 不重写完整用户手册，不覆盖打包发布、云部署或多用户场景。
- 不提交 `config.json`、真实 API Key、私有 Base URL、生成小说正文、向量库数据。
