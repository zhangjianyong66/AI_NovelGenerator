# 重构前端 UI 并对接真实后端接口 - 技术设计

## Scope

本设计覆盖第一个可独立验收的 milestone：项目配置 -> 核心项目文件 -> 章节编辑的真实接口闭环，以及项目页、应用壳层、连接状态的真实数据可信度修复。

本设计不接入真实 LLM 生成执行器，不替换旧 Python GUI，不引入新的后端项目创建能力。

## Design Direction

采用“闭环优先、视觉克制”的方案：

- 保留现有 Tauri/Vue 页面结构、主导航顺序和组件体系。
- 将真实后端状态提升为页面级决策条件：`backend` 模式可读写，`mock` 模式只读预览，`disconnected` 模式显示连接错误。
- 工作台聚焦核心项目文件编辑，章节正文编辑留给独立章节页。
- 隐藏或移除没有真实 API 支撑的主操作入口，例如项目页“新建项目”。
- 保持 mock fallback，但把它明确降级为“离线预览”，禁止保存、导入、生成等写操作。

## Frontend Boundaries

### Service Bridge

`frontend/src/services/serviceBridge.ts` 继续作为唯一服务入口。页面和 store 不直接调用 `mockApi`。

需要补齐的行为：

- 暴露稳定的后端模式读取方式，供页面判断是否可写。
- 读类方法可继续使用 mock fallback。
- 写类方法保持真实后端请求；页面在 mock/断线模式下应提前禁用写按钮，避免用户触发必然失败的保存。
- 错误提示需要包含用户可理解的信息，例如“本地后端未连接，当前为离线预览，不能保存”。

### Project Store

`frontend/src/stores/projects.ts` 需要消除真实后端下默认 mock id 的风险：

- 默认 `activeProjectId` 不应固定为 `p-ember-city`。
- 加载项目后，如果当前 active id 不存在，应选择返回列表的第一个项目。
- 真实后端项目 id 当前为 `current`，页面和后续章节/任务加载应以 store 中的真实 id 为准。

### Editor Store

`frontend/src/stores/editor.ts` 已具备核心文件和章节草稿能力。第一个 milestone 重点是页面调用策略：

- 工作台选择核心文件前，如果当前核心文件有未保存变更，需要确认或阻止切换。
- 核心文件不存在时允许编辑并保存，后端会创建固定文件。
- 章节页只编辑已存在章节；没有章节时显示空状态和准备 `chapter_X.txt` 的说明。
- 章节保存 404 时提示章节文件不存在，不自动创建。

## Page Design

### App Layout

顶部和右侧项目状态栏应准确反映数据来源：

- 后端已连接：显示本地后端已连接。
- mock fallback：显示离线预览，并提示不是当前真实输出目录数据。
- 断线：显示本地后端未连接。

右侧项目状态栏继续展示当前项目、章节进度和章节焦点，但不得在 mock 模式下暗示真实写入能力。

### Projects Page

项目页作为真实配置和项目摘要入口：

- 移除“新建项目”主按钮，避免暗示存在创建项目 API。
- 展示当前输出路径、主题、类型、章节数、每章字数。
- 展示来自 `/api/projects` 的当前项目卡片；如果处于 mock 模式，显示离线预览提示。
- 项目选择只在实际项目列表内切换；真实后端通常只有 `current`。

### Settings Page

第一个 milestone 只要求项目参数页签参与闭环：

- 保存项目配置必须真实写入后端。
- mock/断线模式下禁用保存全部或至少禁用会写入后端的保存动作。
- 输出路径保存后，项目页、工作台、章节页应能刷新并读取新路径的数据。

LLM、Embedding、阶段模型和 WebDAV 页签不作为第一个 milestone 的主验收对象，但不能破坏已有行为。

### Workspace Page

工作台只闭环核心项目文件：

- 左轨保留项目文件列表，章节导航可保留为上下文/跳转线索，但不承担章节正文编辑。
- 中轨只显示核心项目文件编辑器。
- 保存按钮在后端模式可用；mock/断线模式禁用并给出原因。
- 切换核心文件前处理未保存变更。
- 文件不存在时允许用户输入内容并保存创建。
- 右轨展示当前章节摘要、项目上下文、任务状态和日志摘要时，应区分真实后端和离线预览。

### Chapters Page

章节页负责章节正文编辑：

- 已存在章节可编辑和保存。
- 没有章节时显示空状态，说明需要先准备 `chapter_X.txt`。
- 切换章节、上一章/下一章、离开路由时继续保留未保存保护。
- mock/断线模式下章节内容只读预览，保存按钮禁用。
- 保存失败时直接显示后端错误，尤其是章节文件不存在。

### Generation And Knowledge Pages

第一个 milestone 不重构生成任务和知识库主流程，但需要遵守全局写操作策略：

- mock/断线模式下生成、导入、清理向量库、保存角色、写入章节参数等写操作应禁用或明确不可用。
- 生成页应保留“任务只是排队，尚未接入真实 LLM 执行器”的边界提示。

## Validation Design

第一个 milestone 完成后必须验证：

- `cd frontend && npm run typecheck`
- `cd frontend && npm run build`
- 启动本地 FastAPI 和前端 dev server。
- 使用临时输出目录完成浏览器冒烟：
  - 保存项目输出路径。
  - 在工作台编辑并保存核心项目文件，检查真实文件落盘。
  - 在章节页编辑并保存已存在章节，检查真实文件落盘。
  - 停止后端或模拟后端不可用，确认页面进入离线预览或断线状态，写按钮不可用。

## Rollback

回滚策略保持简单：

- 前端改动集中在 `frontend/src/services/`、`frontend/src/stores/`、`frontend/src/pages/`、`frontend/src/layouts/` 和少量通用 UI 状态组件。
- 不改变后端数据文件格式。
- 不改变 `config.json` schema。
- 若 mock/后端状态策略引入问题，可先回退页面禁用态和提示层，不影响后端接口。
