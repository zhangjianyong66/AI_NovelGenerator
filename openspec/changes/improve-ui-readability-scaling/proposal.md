## Why

当前桌面 GUI 在高分辨率或大尺寸窗口下整体文字偏小，右侧配置区、标签页、按钮和文本输入控件可读性不足。用户已确认采用“舒适版”方案，需要在不重做信息架构的前提下提高全局可读性。

## What Changes

- 引入统一的 UI 字体与缩放约定，避免继续分散维护大量硬编码字号。
- 将常规控件字体从现有 12 左右提升到舒适阅读尺寸。
- 将主要长文本编辑/日志区域字体略大于普通控件，提升写作和校对体验。
- 设置 CustomTkinter 控件缩放，使按钮、输入框、标签页等控件尺寸随字体同步变得更易点击和阅读。
- 必要时调整主窗口右侧配置区域宽度，避免舒适版字号造成挤压。
- 不改变应用功能流程、配置结构、LLM 调用逻辑或输出文件格式。

## Capabilities

### New Capabilities
- `ui-readability-scaling`: 定义桌面 GUI 的舒适版全局字体、控件缩放和可读性要求。

### Modified Capabilities

无。

## Impact

- 主要影响 `main.py` 和 `ui/` 下 CustomTkinter 界面构建代码。
- 可能新增一个集中式 UI 样式模块，用于统一字体常量和缩放参数。
- 不新增运行时依赖，不改变持久化配置文件格式。
- 验证重点为应用可正常启动、主要页面控件不明显重叠、文字可读性提升。
